# Training data shards (v0)

Layout for the checked-in **training slice** used by decoder v0 and CI. Heavy datasets live outside this repo; this shard is the minimal reproducible contract.

**Related:** [Z_BUILD.md](Z_BUILD.md) (`z` recipe), [VECTORBENCH_V0.md](VECTORBENCH_V0.md) (oracle tasks), [TRAINING_ON_MAC.md](TRAINING_ON_MAC.md) (external training repo layout), [TRUST_AND_CANONICAL_ARTIFACTS.md](TRUST_AND_CANONICAL_ARTIFACTS.md) (manifest philosophy).

---

## Shard root

Default path (generated by [`scripts/gen_training_rows.py`](../scripts/gen_training_rows.py)):

```
benchmarks/datasets/training_slice_v0/
  manifest.json      # dataset_manifest v1 — sha256 index
  rows.jsonl         # one training row per line
  programs/          # .vcir copies (paths referenced from rows)
```

Regenerate:

```bash
python3 scripts/gen_training_rows.py --write
bash scripts/verify-dataset-manifest.sh benchmarks/datasets/training_slice_v0 \
  benchmarks/datasets/training_slice_v0/manifest.json
```

`--verify-only` checks rows and manifest without writing; `--count N` adds synthetic `program_id`s for stress tests (see script `--help`).

---

## `rows.jsonl` fields

Each line is one JSON object (schema: [`schemas/training_shard_v1.schema.json`](../schemas/training_shard_v1.schema.json)).

| Field | Type | Meaning |
|-------|------|---------|
| `program_id` | string | Stable key; **`z` is derived from this** ([Z_BUILD.md](Z_BUILD.md)) |
| `task_id` | string | VectorBench `task_id` (see mapping below) |
| `split` | string | `train` \| `val` \| `test` |
| `z` | `[f32]` × 256 | Frozen latent for this row |
| `vcir_path` | string | `.vcir` **relative to shard root** (e.g. `programs/add.vcir`) |
| `manifest` | string | Shard checksum index, usually `manifest.json` (relative to shard root) |

Optional fields may be added in a new schema version; do not rename required fields without bumping `dataset_version`.

Regenerate the checked-in shard after changing programs or the Z_BUILD recipe:

```bash
python3 scripts/gen_training_rows.py --write
```

## Splits

**v0 canonical slice (3 programs):** one row per program; split is **`train`** for all (memorization / plumbing). The generator may assign:

| `program_id` | `task_id` | Typical `split` |
|--------------|-----------|-----------------|
| `add` | `add_i32` | `train` |
| `mul` | `mul_i32` | `train` |
| `const42` | `const42_i32` | `train` |

**Larger datasets** (external repo or `--count N`):

- Prefer **split by `program_id`**, not by opcode or case — all rows for one program stay in one split (80/10/10 train/val/test in [TRAINING_ON_MAC.md](TRAINING_ON_MAC.md)).
- **Prefix rule (optional):** e.g. `holdout_` prefix on `program_id` → `split: "test"` if you do not store an explicit column.
- **Explicit column (recommended):** set `split` in `rows.jsonl` at generation time; no ambiguity.

Never tune generator knobs on **test** / holdout programs.

## `task_id` → manifest mapping

Authoritative list: [`benchmarks/vectorbench_v0/suite.json`](../benchmarks/vectorbench_v0/suite.json).

| `task_id` | Bench manifest (repo root) | Canonical `.vcir` family |
|-----------|--------------------------|---------------------------|
| `add_i32` | `benchmarks/manifests/add.json` | `add` |
| `mul_i32` | `benchmarks/manifests/mul.json` | `mul` |
| `const42_i32` | `benchmarks/manifests/const42.json` | `const42` |

Shard rows point at `programs/{family}.vcir` under the slice; golden sources also live under `benchmarks/programs/`.

Training loops should call the oracle with **the row’s `task_id`**, not a global default. Metrics: [VECTORBENCH_V0.md](VECTORBENCH_V0.md).

## External training repo + `VECTORC_ROOT`

Training code should **not** copy oracle logic. Point at this workspace:

```bash
export VECTORC_ROOT=/path/to/VectorComplier
```

| Need | Use |
|------|-----|
| Fast metric for one `.vcir` | `python3 "$VECTORC_ROOT/scripts/vectorbench_oracle.py" --vcir … --task "$task_id" --metric execute_rate` |
| Decode + eval | `bash "$VECTORC_ROOT/scripts/training-oracle.sh" --z … --vcir …` (set `DECODER`, `ONNX_MODEL` for onnx) |
| Full CI path | `"$VECTORC_ROOT/scripts/eval-decode-scoreboard.sh"` — [DEVELOPMENT.md](DEVELOPMENT.md) |
| Batch shard eval / failure dumps | `batch-eval-training-shard.sh`, `dump-eval-failures.sh` — [DEBUGGING_DECODE.md](DEBUGGING_DECODE.md) |

`vectorbench_oracle.py` runs `cargo run -p vc-cli -- eval` with `cwd=$VECTORC_ROOT`. Override with `--vectorc` / `--vectorc-root` if needed.

Load shards from the training repo copy or submodule path; keep **`z` bytes identical** to [Z_BUILD.md](Z_BUILD.md) when writing `z.json` for `decode-z`.

## Checksum manifest workflow

Shard `manifest.json` follows [`schemas/dataset_manifest_v1.schema.json`](../schemas/dataset_manifest_v1.schema.json):

- Lists every `.vcir`, `rows.jsonl`, and other artifacts under `artifacts[]` with **`relative_path`** + **`sha256`** (paths relative to the shard directory passed as `DATASET_ROOT`).
- Optional `splits` metadata for humans; row-level `split` in `rows.jsonl` is authoritative for training code.

Verify after any edit or regeneration:

```bash
bash scripts/verify-dataset-manifest.sh benchmarks/datasets/training_slice_v0 \
  benchmarks/datasets/training_slice_v0/manifest.json
```

CI runs the same check on [`fixture_slice_v0`](../benchmarks/datasets/fixture_slice_v0/manifest.json) — see [`.github/workflows/ci.yml`](../.github/workflows/ci.yml). Preflight: [`scripts/preflight.sh`](../scripts/preflight.sh).

To refresh checksums after changing programs, use `gen_training_rows.py --write` or update `sha256` fields via the generator’s manifest output.

## Failure analysis

When `vectorc eval` reports `ok=false` for a decoded `.vcir`, capture artifacts before re-running decode. Full workflow: [DEBUGGING_DECODE.md](DEBUGGING_DECODE.md).

**Single sample** — writes `summary.json`, `errors.txt`, `program.vcir`, optional `inspect.json`:

```bash
bash scripts/dump-eval-failures.sh \
  --vcir /tmp/decoded.vcir \
  --task add_i32 \
  --out /tmp/fail-001

# Include IR inspect summary
bash scripts/dump-eval-failures.sh --vcir /tmp/decoded.vcir --task add_i32 \
  --out /tmp/fail-001 --inspect
```

**Whole shard** — aggregate JSON + `failures` list (`.vcir` paths where `ok=false`):

```bash
bash scripts/batch-eval-training-shard.sh \
  benchmarks/datasets/training_slice_v0 \
  --out /tmp/shard-report.json

# Per-failure dump directories under /tmp/fail-dumps/<row_id>/
bash scripts/batch-eval-training-shard.sh benchmarks/datasets/training_slice_v0 \
  --dump-failures /tmp/fail-dumps --out /tmp/shard-report.json
```

Set `VECTORC=./target/debug/vectorc` (or install `vectorc` on `PATH`) to skip `cargo run` during triage.
